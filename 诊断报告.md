## 3. 错误处理

### 现状
| 场景 | 现状 |
|------|------|
| **LLM 调用失败** | ✅ llm.py 中 `_LiteLLMClient.chat()`/`chat_stream()` 捕获 `Exception` 包装为 `LLMError`；`_DeepSeekClient` / `_OllamaClient` 同样处理 |
| **LLM 错误传播** | ✅ engine.py 中 `process_with_result` 捕获 `LLMError` 返回 `[系统提示] {e}`（不会崩溃）；`process_stream` 同样处理 |
| **API Key 缺失** | ✅ cli.py 中 `_confirm_api()` 在 chat/ask 前检查；`_check_config()` 返回 warning 但不阻止 |
| **用户 Ctrl+C** | ✅ `_interactive_tui` 中 `PromptSession` 绑定了 Ctrl+C 键（KeyboardInterrupt）；`_run_dialectic_tui` 的 `session_ps.prompt_async` 也捕获 `EOFError/KeyboardInterrupt` |
| **用户知识库不存在** | ✅ `config.py` 中 `get_knowledge_roots()`、`knowledge list` 等都有 path exists 检查 |
| **配置不存在** | ✅ `QiushiConfig.load()` 返回默认值 |
| **HTTP 超时/连接错误** | ⚠️ 仅在 WebSearcher 中捕获 `Exception` 并 log debug。`httpx.AsyncClient` 在 `_DeepSeekClient` 中没有专属超时重试或连接失败处理 |

### 问题 — 必须修复
1. **❌ `_DeepSeekClient.chat_stream()` 每次调用都新创建 `httpx.AsyncClient`** — 行 142 每次流式调用都重新构造 client，不共享连接池。这浪费连接且没有复用。
2. **❌ `process_stream` 中的 `except LLMError` 后 yield 了错误消息，但 `stream: bool` 调用方完全不知道是错误还是正常回复** — 没有错误信号通道。
3. **❌ 整个项目没有连接超时重试逻辑** — `httpx` client 的 `timeout` 设置在 config 中只有 120s 读超时，但连接超时 (`connect_timeout`) 未显式设置，依赖 httpx 默认值。网络闪断场景没有重试。

### 问题 — 可选优化
- `_LiteLLMClient` 中 `os.environ.setdefault` 设置 API key 是进程全局副作用，建议用 litellm 的 api_key 参数。

## 总结优先级

### 🔴 必须修复 (按影响排序)
1. **cli.py `knowledge add` 死代码** — 28 行不可达代码，功能不可用。修复 `if not action or action not in ("add", "list", "reload"):` 条件逻辑（第 843 行）。
2. **`engine.py` typo "matced"** — 概念炼金术在 --explain 输出中静默丢失。修复 analyzer.py 和 engine.py 中 `matced_concepts` → `matched_concepts`。
3. **cli.py 和 tui/ 零测试覆盖** — 至少为 `chat`, `ask`, `init`, `knowledge` 等 CLI 入口添加 mock 集成测试。
4. **`_DeepSeekClient.chat_stream()` 每次新开 httpx client** — 应复用 `self._client` 或至少同一 config 生命周期内的 client。
5. **`knowledge add` CLI 功能在 `action not in ("add", "list", "reload")` 条件下被错误短路** — 它被当作"无效"动作，没有执行添加逻辑。

- 添加 lock 文件锁定依赖
- 去除 `duckduckgo_search`、`beautifulsoup4` 虚假依赖
- 添加 `QiushiConfig` 的 schema 验证
- `init` 后增加连通性验证
- 网络错误重试逻辑
- 常量抽取 magic number
